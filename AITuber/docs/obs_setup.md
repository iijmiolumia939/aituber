# OBS セットアップガイド

AITuber 配信のための OBS Studio 設定手順です。

## 自動セットアップ（推奨）

```powershell
# シーンコレクション・プロファイル・global.ini を一括生成
python tools/setup_obs.py --force

# そのまま OBS を起動
python tools/setup_obs.py --force --launch
```

これで以下がすべて自動設定されます。手動設定は不要です。

## 前提条件

| ツール | バージョン | 用途 |
|--------|-----------|------|
| OBS Studio | 32.x 以上 | 配信ソフト |
| AITuber Unity ビルド | `Build/AITuber.exe` | アバター描画（グリーンバック） |
| Python オーケストレーター | `orchestrator/` | LLM・TTS・WS 制御 |

## アーキテクチャ概要

```
┌──────────────┐   WS :31900   ┌───────────────────┐
│  Python      │ ─────────────→│  Unity (avatar)   │
│  Orchestrator│               │  グリーンバック     │
│              │   WS :31901   │                   │
│              │ ──┬──────────→│                   │
└──────────────┘   │           └───────────────────┘
                   │                     │
                   │               ゲームキャプチャ
                   ▼                     ▼
         ┌───────────────┐      ┌───────────────┐
         │ OBS ブラウザ   │      │ OBS クロマキー │
         │ ソース (HTML)  │      │ フィルタ       │
         └───────────────┘      └───────────────┘
```

- **Unity** — グリーンバック (RGB: 0,255,0) でアバターを描画
- **オーケストレーター** — `:31901` でオーバーレイ用 WebSocket サーバーを起動
- **OBS** — ゲームキャプチャ＋クロマキーでアバターを抜き、ブラウザソースで HTML オーバーレイを重ねる

---

## 1. OBS シーンの作成

OBS を起動し、新しいシーンを作成します（例: `AITuber 配信`）。

---

## 2. アバター（ゲームキャプチャ）

### ソース追加
1. **ソース** → **＋** → **ゲームキャプチャ**
2. 名前: `Avatar`
3. モード: **特定のウィンドウをキャプチャ**
4. ウィンドウ: `AITuber` (Unity ビルドを起動済みの状態で選択)
5. **透過を許可** にチェック

### クロマキーフィルタ
1. `Avatar` ソースを右クリック → **フィルタ**
2. **エフェクトフィルタ** → **＋** → **クロマキー**
3. 設定:
   - 色キーの種類: **緑**
   - 類似性 (Similarity): `400` (初期値, 必要に応じて調整)
   - 滑らかさ (Smoothness): `80`
   - キーカラー流出低減 (Key Color Spill Reduction): `100`
4. プレビューでアバター周囲の緑が消えることを確認
5. 必要に応じてソースを移動・リサイズ（推奨: 画面右下〜中央）

---

## 3. HTML オーバーレイ（ブラウザソース）

オーケストレーターが `:31901` で WebSocket サーバーを起動します。  
HTML ファイルはローカルファイルとして指定します。

- 配置: 画面上端 `(0, 0)`
- LIVE バッジ（赤→ピンクグラデーション）、視聴者/登録者数パネル、配信タイトル、時計を表示

### 3.2. チャット (`chat.html`)

| 項目 | 設定値 |
|------|--------|
| ソース種別 | ブラウザ |
| 名前 | `Chat` |
| ローカルファイル | ✅ チェック |

### 3.3. 字幕 (`subtitle.html`)

|------|--------|

## 4. 推奨レイヤー順序（上が前面）

```
┌─────────────────────────────┐
│ 4. Subtitle   (最前面)       │
│ 3. Chat                     │
│ 2. Header                   │
│ 1. Avatar                   │
│ 0. Background (最背面)       │
```

OBS のソースリストで上にあるほど前面に描画されます。  
`Background` はダークネイビー (#1a1a33) のカラーソースで、アバター未起動時の黒画面を防ぎます。

## 6. 起動手順

```powershell
# 1. Unity アバターを起動 (グリーンバック)
Start-Process "Build/AITuber.exe"

# 2. YUI.A としてオーケストレーターを起動 (WS :31900 + :31901)
python tools/launch.py -c yuia

# または .env に CHARACTER_NAME=yuia が設定済みであれば
python tools/launch.py

# 3. OBS を起動してシーンを選択 → 配信開始
```

> **Note**:
> - `.env` に `CHARACTER_NAME=yuia` が設定されていれば `-c yuia` は省略可能です。
> - Python オーケストレーターが起動すると `:31901` の WebSocket サーバーが立ち上がります。
> - OBS のブラウザソースは自動的に接続を試みるため、起動順序は厳密でなくても問題ありません。
> - ブラウザソースが「接続待ち」になった場合は、ソースを右クリック → **再読み込み** で再接続されます。
> - `LIVE_CHAT_ID` は起動時に YouTube API から自動取得されます (M16)。  
>   `YOUTUBE_CHANNEL_ID` が `.env` に設定されていない場合は `LIVE_CHAT_ID` を手動で `.env` に記載してください。

## 7. トラブルシューティング

| 症状 | 対処 |
|------|------|
| アバターの緑が残る | クロマキーの「類似性」値を上げる (400→500) |
| アバターの輪郭が欠ける | 「滑らかさ」を下げる (80→40) |
| オーバーレイが表示されない | Python 起動確認 → ブラウザソースを右クリック→再読み込み |
| オーバーレイの文字化け | ブラウザソースの「カスタム CSS」が空であることを確認 |
| チャットが流れない | YouTube Chat Poller のログで polled 件数を確認 |
| 字幕が消えない | `subtitle.html` の duration_sec を確認。手動テスト: WS クライアントで `{"type":"subtitle","text":"","duration_sec":0}` 送信 |

---

## 8. （任意）背景の追加

アバターの背後に背景画像や動画を追加する場合:

1. **ソース** → **＋** → **画像** or **メディアソース**
2. `Avatar` ソースの **下** にレイヤー配置
3. クロマキーで抜かれたアバターが背景の上に表示されます
