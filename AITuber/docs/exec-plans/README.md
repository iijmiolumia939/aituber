# exec-plans — 実行計画ディレクトリ

> 複雑な実装タスク（3日以上 / 複数サブシステムにまたがる）は  
> このディレクトリに **実行計画ファイル** を作成してから着手する。  
> 進捗ログ・設計決定を逐次記録し、完了後 `completed/` に移動する。

---

## ファイル構成

```
exec-plans/
├── README.md          このファイル（フォーマット仕様）
├── active/            進行中の計画
│   └── m2-reflection-runner.md
└── completed/         完了済み計画（変更不可）
    └── m1-gap-logger.md
```

---

## 実行計画テンプレート

新しい計画は以下のテンプレートをコピーして `active/<name>.md` を作成する。

```markdown
# [計画名] exec-plan

> **作成**: YYYY-MM-DD  
> **目標**: （1〜2文で何を達成するか）  
> **関連**: SRS FR/NFR ID, GitHub Issue, PLANS.md 行  
> **依存**: （他の計画・PR など）

---

## ゴール

（Why: なぜこれが必要か。Done の定義。）

---

## スコープ

| 含む | 含まない |
|---|---|
| ... | ... |

---
```

---

## ガイドライン

- **小さく始める**: 計画は最初から完璧を目指さない。タスクは発見次第追加。
- **決定を残す**: 却下した設計案・理由も Design Decision Log に書く。失われた文脈はエージェントに伝わらない。
- **進捗は毎日更新**: 作業したら必ず進捗ログに追記する。
- **「計画が現実と違う」は正常**: 計画を実態に合わせて修正することは歓迎。計画と実装の乖離放置は禁止。
